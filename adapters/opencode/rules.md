# SpecLane rules for OpenCode

Follow the SpecLane loop for non-trivial work: `specify → plan → tasks → implement → verify →
archive`, inside a change folder under `changes/`, using `templates/` and `sdd/constitution.md`.
Pick a track from `docs/tracks.md`; approval domains require human review. The spec is the
source of truth; humans own merges and contract changes; no secrets; tests ship with behavior.
