# FAQ

**Is SpecLane a language or a framework to install?**
Neither in the runtime sense. It's a methodology plus a small Bash CLI and a set of Markdown
conventions. Nothing to compile; no runtime dependency in your app.

**Does it replace my agent or my IDE?**
No. It sits on top of whatever agent you use and gives it a consistent process.

**Does it lock me into a model provider?**
No. Provider config is isolated in `config/`. Anthropic, OpenAI-compatible, and z.ai are all
first-class; switching is a config change, not a rewrite.

**Isn't this just more paperwork?**
The artifacts are sized to risk, and most evidence is generated by doing the work (tests, PR
descriptions). A `quickfix` is a task + PR + check. The heavy path only applies to risky work.

**Do I need all the directories?**
No. Keep `sdd/`, `templates/`, `changes/`, `specs/`, `scripts/`, and the one adapter you use.
The rest is opt-in.

**Can a team and a solo dev both use it?**
Yes. Roles are accountabilities, not headcount. Solo devs wear all hats; the process still keeps
intent and evidence linked.

**Where do secrets go?**
In `.env` (gitignored). Never in specs, commits, prompts, or logs.
